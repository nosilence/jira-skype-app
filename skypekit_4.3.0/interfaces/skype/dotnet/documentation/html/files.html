Here is a list of all documented files with brief descriptions:
skt_baseclasses.cs [code]Contains abstract parent classes for all the classes in the skt_skypekit.cs The SktObject is the common ancestor to all the SkypeKit classes - for the purpose of being able to typecast all the objects to the common root. SktSkypeBase is the special ancestor of the SktSkype class - containing everything that did not make sense to put in the script-generated skt_skypekit.cs Similarly, the SktEventsBase class implements everything for events, that is specific to .NET wrapper, rather than abstract SkypeKit API
skt_skypekit_2010.cs [code]This is the main module of the .NET wrapper, that contains all the user-servicable SkypeKit API classes, plus a special SktEvents class for events. NB! This is a script-generated file. Manual modifications will be lost
skt_videorenderer.cs [code]Contains the SktVideoRenderer video renderer class (descended from PictureBox) and the entirety of the client side of the shared memory based frame transport mechanism, in two classes: FrameTransport - implements the frame transport IPC, including marshalling between .NET managed memory and shared memory. SharedMemoryChannel - implements Windows based mapping mechanism to shared memory resources. NB! As this is making liberal use of Win API functions imported from Kernel32.dll, it is somewhat unlikely to work too well with Mono
